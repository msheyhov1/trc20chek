# План деплоя на Railway

Это пошаговая инструкция для Claude Code. Иди строго по шагам, **жди подтверждения пользователя после каждого**. Не пропускай шаги и не запускай несколько команд параллельно — пользователь должен видеть, что происходит.

## Контекст

- Проект уже распакован в текущей папке (см. `CLAUDE.md` и `README.md`)
- Пользователь работает на macOS (MacBook Air)
- GitHub-аккаунт у пользователя уже есть
- Цель: задеплоить проект на Railway так, чтобы работали REST API, Telegram-бот и веб-форма

## Задача 3. Инициализировать git и сделать первый коммит

**Что сделать (подставив URL из задачи 2):**

```bash
# Проверить что мы в правильной папке
pwd
ls Dockerfile  # должен существовать

# Убедиться что .env НЕ закоммитится (если он вдруг есть локально)
cat .gitignore | grep -E "^\.env$"

# Инициализация
git init
git add .
git status  # показать что собираемся закоммитить
```

**Перед коммитом проверь вывод `git status`:**
- В списке НЕ должно быть файла `.env` (только `.env.example`)
- В списке НЕ должно быть `cache.db`, `__pycache__/`, `.pytest_cache/`

Если что-то из этого есть — останови, разберись почему `.gitignore` не сработал.

**Если всё чисто, продолжай:**

```bash
git commit -m "Initial commit: TRC20 checker for Railway"
git branch -M main
git remote add origin <URL_ОТ_ПОЛЬЗОВАТЕЛЯ>
git push -u origin main
```

**Если push потребует аутентификации:**
- GitHub с 2021 года не принимает пароль аккаунта
- Варианты:
  1. **Personal Access Token (PAT)** — пользователь создаёт на https://github.com/settings/tokens (classic, scope `repo`), вводит вместо пароля
  2. **GitHub CLI** — `brew install gh && gh auth login`
  3. **SSH-ключ** — если у пользователя уже настроен, нужно поменять remote на SSH-вариант

Спроси пользователя что у него уже есть, и иди по самому короткому пути.

**Критерии готовности:**
- `git push` завершился без ошибок
- На странице репозитория в GitHub видны все файлы проекта
- Видно `CLAUDE.md`, `README.md`, папки `core/`, `api/`, `bot/`, `web/`
- НЕ видно `.env` файла

**После выполнения:** попроси пользователя открыть страницу репозитория и подтвердить что все файлы залились. Особенно подчеркни — нужно убедиться что `.env` НЕ виден на GitHub.

## Задача 6. Задать переменные окружения

**Это делает пользователь, ты пошагово диктуешь:**

Скажи пользователю:

> В дашборде Railway-сервиса:
> 1. Открой вкладку **Variables**
> 2. Нажми **+ New Variable**
> 3. Добавь по одной (каждая отдельно):
>
> | Имя | Значение |
> |---|---|
> | `BOT_TOKEN` | <токен от @BotFather, который ты получал раньше> |
> | `CACHE_TTL_SECONDS` | `604800` |
>
> `TRONSCAN_API_KEY`, `GOPLUS_API_KEY`, `API_KEY` — пока пропусти, не обязательно.
>
> После добавления Railway автоматически перезапустит сервис.

**Если у пользователя нет под рукой токена бота:**
- Напомни что его выдавал @BotFather в Telegram
- Если потерян — пусть пишет `/mybots` боту BotFather, выбирает своего бота, **API Token** → **Revoke current token** → получит новый
- Если бота вообще не создавал — пусть создаст через `/newbot` (см. шаг 1 в README.md)

**Критерии готовности:**
- В Variables видно `BOT_TOKEN` и `CACHE_TTL_SECONDS`
- В логах Railway виден старт без ошибок (см. задачу 8)

## Задача 7. Добавить persistent volume

**КРИТИЧНО.** Без этого SQLite-кеш будет теряться при каждом деплое.

Скажи пользователю:

> В дашборде сервиса:
> 1. Открой вкладку **Settings**
> 2. Прокрути до раздела **Volumes** (может называться "Storage" или "Disk")
> 3. Нажми **+ New Volume**
> 4. **Mount path:** `/data`
> 5. Размер: 1 GB (минимально доступный)
> 6. Save / Create
>
> Railway перезапустит сервис после добавления тома.

**Критерии готовности:**
- В Settings → Volumes виден один том с mount path `/data`

## Задача 8. Проверить логи

Скажи пользователю открыть вкладку **Deployments** → последний деплой → **View Logs**.

В логах должно быть:

```
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:app:Starting Telegram bot polling...
INFO:aiogram.dispatcher: Start polling
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:XXXX
```

**Если есть ошибки:**

| Ошибка в логах | Что делать |
|---|---|
| `BOT_TOKEN not set — Telegram bot disabled, API-only mode` | Вернуться к задаче 6, добавить `BOT_TOKEN` |
| `aiogram.exceptions.TelegramUnauthorizedError` | Токен невалиден — проверить значение `BOT_TOKEN`, нет ли пробелов, не отозван ли |
| `TelegramConflictError: terminated by other getUpdates request` | У пользователя где-то ещё запущен бот с тем же токеном. Спросить — может локально или на другом сервисе. Остановить второй экземпляр |
| `sqlite3.OperationalError: unable to open database file` | Volume не подключён или mount path не `/data`. Вернуться к задаче 7 |
| Другая ошибка | Скопировать последние 30 строк логов и обсудить с пользователем |

**После починки** Railway автоматически передеплоит. Дождись повторной проверки логов.

## Задача 10. Финальная проверка

Когда есть URL, выполни через curl три проверки:

```bash
# Health check
curl https://URL_ОТ_ПОЛЬЗОВАТЕЛЯ/health

# Должно вернуть: {"status":"ok","bot_enabled":true}
```

```bash
# Проверка реального адреса (USDT contract)
curl https://URL_ОТ_ПОЛЬЗОВАТЕЛЯ/check/TR7NHqjeKQxGTCi8q8ZY4pL8otSzgjLj6t
```

```bash
# Проверка невалидного адреса
curl -i https://URL_ОТ_ПОЛЬЗОВАТЕЛЯ/check/INVALID

# Должно вернуть HTTP 400 + {"detail":"Invalid TRC20 address format"}
```

**Также:**
- Открой `https://URL/` в браузере — должна показаться веб-форма
- Попроси пользователя написать своему боту в Telegram любой TRC20-адрес и проверить ответ

**Критерии готовности (всё одновременно):**
- ✅ `/health` возвращает `bot_enabled: true`
- ✅ `/check/<valid_addr>` возвращает JSON с полями `entity`, `risk_level`, `sources`
- ✅ `/check/INVALID` возвращает 400
- ✅ Веб-форма открывается и работает (можно вставить адрес и нажать "Проверить")
- ✅ Telegram-бот отвечает на адрес verdict'ом

## Если что-то идёт не так

1. **Билд падает с ошибкой Python/pip** — покажи последние 50 строк билд-логов и обсуди с пользователем
2. **Healthcheck не проходит** — проверь что в `railway.json` `healthcheckPath: "/health"` совпадает с реальным эндпоинтом в `api/main.py`
3. **Бот не отвечает, но логи чистые** — пользователь может писать не тому боту. Пусть найдёт своего по username, который вводил при создании у @BotFather

## Что НЕ делать

- Не предлагай локальный запуск (Docker Desktop, uvicorn) — пользователь хочет именно на Railway
- Не предлагай переходить на Fly.io/Render — выбор уже сделан
- Не меняй структуру проекта без явной просьбы — она уже отлажена
- Не меняй `docker-compose.yml` (его нет в этой версии) и не добавляй обратно — Railway его не использует
- Не запускай `pytest` в Railway-окружении — тесты гоняются локально, не в деплое
